# Concept: Security Awareness and Education

## Overview
Awareness, education, and training are distinct steps that build on each other to influence security behavior.

## The Three Steps
1. **Security Awareness**: Catching attention and convincing people that security is relevant and that they are capable of taking action.
2. **Security Education**: Providing knowledge about risks to build accurate **Mental Models**.
3. **Security Training**: Acquiring specific skills (e.g., using a security tool, recognizing social engineering).

## Behavioral Change Model (RISCS)
Knowing what to do is not enough. Behavioral change requires:
- **Awareness** (The first step)
- **Understanding**
- **Motivation**
- **Ability**
- **Trigger/Prompt** (Fogg Behavior Model)
- **Embedded Habits**: Replacing old, automatic routines with new security-conscious ones.

## Fogg Behavior Model
Behavior (B) happens when **Motivation (M)**, **Ability (A)**, and a **Prompt (P)** converge at the same moment ($B=MAP$).
- If a person has high motivation but low ability, they will get frustrated.
- If they have high ability but low motivation, they will ignore the task.
- A prompt (like a security warning) only works if the user is above the "action line" for both.

## Modern Approaches
- **Simulations**: Anti-phishing tests can create "teachable moments," but must be handled carefully to avoid damaging trust.
- **Games**: Used to make learning more engaging and to allow experimentation in a safe environment.
- **Contextual Learning**: Training is most effective when it happens close to the task it relates to.
