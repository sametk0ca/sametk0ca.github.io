#### **HASH VALUES**

As per Microsoft, the hash value is a numeric value of a fixed length that ==uniquely identifies data==.

common hashing algorithms:

- MD5
- SHA1
- SHA2

Various online tools can be used to do hash lookups like [VirusTotal](https://www.virustotal.com/gui/) and [Metadefender Cloud - OPSWAT](https://metadefender.opswat.com/?lang=en).

#### **IP ADDRESS**

 An IP address is used to ==identify any device connected to a network==. These devices range from desktops, to servers and even CCTV cameras.

From a defense standpoint, knowledge of the IP addresses an adversary uses can be valuable. A common defense tactic is to block, drop, or deny inbound requests from IP addresses on your parameter or external firewall. This tactic is often not bulletproof as it’s trivial for an experienced adversary to recover simply by using a new public IP address.

#### **DOMAIN NAMES**

Domain Names can be thought as ==simply mapping an IP address to a string of text.== 

A domain name can contain a domain and a top-level domain (e.g. `evilcorp.com` ) or a sub-domain followed by a domain and top-level domain (e.g. `tryhackme.evilcorp.com`)
